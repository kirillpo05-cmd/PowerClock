## 1. Проблема

Цена электричества в Латвии (биржа Nord Pool, день-вперёд) меняется **каждый час** и
колеблется в разы: в типичный день разброс между самым дешёвым и самым дорогим часом —
**3–10×** (например, 2 ц/кВт·ч ночью против 15–25 ц/кВт·ч в вечерний пик; в пиковые дни
и выше). Для дома с электроплитой/бойлером/зарядкой авто перенос нагрузки в дешёвые часы
экономит десятки евро в месяц.

При этом существующие способы посмотреть цену:
- биржевые таблицы и «свечные» графики — нечитабельны для обычного человека;
- приложения энергокомпаний — показывают «сейчас/завтра», без паттернов
  («какой час суток стабильно дешёвый?», «какой день недели дороже?»);
- никто не показывает **цикличность** данных — а именно она и есть главная структура:
  сутки, неделя, год.

**Боль в одной строке:** почасовые цены цикличны по природе, но все показывают их
плоскими графиками; красивого инструмента «увидеть циклы и понять, когда включать
стиралку» — нет.

## 2. Решение

Одностраничный аналитический сайт **PowerClock**: радиальные («круговые») визуализации
почасовых цен Nord Pool для Латвии. Каждый шаг — модуль:

1. **Data pipeline** — GitHub Actions по крону (раз в день) тянет цены из открытого
   API Elering (Nord Pool day-ahead, зоны LV/EE/LT, JSON без ключа), нормализует и
   коммитит снэпшоты `data/*.json` в репозиторий. → *модуль `pipeline`*
2. **Price Clock (главный экран)** — «циферблат» на 24 часа: радиальная колонковая
   диаграмма сегодняшних (и завтрашних, когда опубликованы) цен; текущий час подсвечен
   стрелкой; цветовая шкала дёшево→дорого. → *модуль `clock`*
3. **Weekly Ring** — кольцевая тепловая карта: 7 колец (дни недели) × 24 сектора
   (часы), средние цены за последние N недель — паттерн «дешёвая ночь / дорогой
   вечер будня» виден с одного взгляда. → *модуль `weekring`*
4. **Year Spiral** — спираль из 12 месяцев (или 52 недель): сезонность года.
   → *модуль `spiral`*
5. **Insight-карточки** — 3–4 автоматических вывода из данных: «самый дешёвый час
   сегодня: 03:00 (2.1 ц)», «дешевле всего на неделе: вс днём», «сегодня на 18%
   дороже среднего за месяц». → *модуль `insights`*
6. **Витрина** — тёмная тема, аккуратная типографика, плавные анимации входа,
   tooltips; README со скриншотами/GIF. → *сквозное требование, не модуль*

## 3. Почему сейчас

- Цены на электричество — **горячая тема** в Латвии/Балтии; проект мгновенно понятен
  любому местному рекрутёру («о, это про наши счета»).
- API Elering — открытый, бесплатный, без регистрации: почасовые цены NPS по зонам
  LV/EE/LT в JSON. Редкий случай качественных открытых данных без ключей и квот.
- GitHub Actions (бесплатный cron) + GitHub Pages (бесплатный хостинг) позволяют
  сделать «живой, самообновляющийся» сайт с нулевой инфраструктурой.
- В портфолио кандидата нет чисто **визуализационного** проекта — этот закрывает нишу
  «фронтенд + данные + вкус».

## 4. Целевая аудитория

**Основная:** рекрутёры и разработчики (Magebit, MIM, фронтенд/data-вакансии), открывшие
ссылку на 1–3 минуты. Задача — впечатление: «красиво, данные настоящие, сделано
аккуратно». Следствие: первый экран должен «продавать» без скролла и объяснений.

**Вторичная:** реальные жители Латвии, планирующие энергоёмкие дела (стирка, бойлер,
зарядка EV). Следствие: минимум один практический элемент — «дешёвые часы сегодня/завтра».

## 5. Архитектура

### Слои

```
GitHub Actions (cron, 1×/день ~15:00 EET — после публикации цен на завтра)
   │  fetch Elering API (LV) → normalize → data/daily/YYYY-MM-DD.json
   │                                      → data/aggregates.json (недельные/годовые агрегаты)
   ▼  git commit + push
GitHub Pages (статика)
   ▲
   │  build (Vite) при пуше в main
React + TypeScript SPA
   ├── DataLoader  — грузит data/*.json (fetch), парсит, кэширует
   ├── ClockChart  — D3: радиальные бары 24h (сегодня/завтра, переключатель)
   ├── WeekRing    — D3: кольцевая heatmap 7×24
   ├── YearSpiral  — D3: спираль месяцев
   ├── Insights    — чистые функции-агрегаты → карточки
   └── UI shell    — layout, тёмная тема, легенда, tooltips
```

### Стек с обоснованием

| Слой | Технология | Почему |
|---|---|---|
| Данные | Elering API (Nord Pool day-ahead, зона LV) | Открытый JSON без ключа; почасовые цены; официальный источник |
| Пайплайн | GitHub Actions (cron) + Node/Python-скрипт | Бесплатный планировщик; данные версионируются в git |
| Frontend | React 18 + TypeScript + Vite | Твой рабочий стек; быстрый билд |
| Визуализация | **D3.js** (svg) | Радиальные диаграммы — кастом; готовые либы (recharts) их толком не умеют. D3 в резюме весит много |
| Стили | Tailwind CSS + CSS-переменные темы | Быстро и консистентно |
| Хостинг | GitHub Pages | Бесплатно, деплой из Actions |
| Тесты | Vitest — юнит на агрегаты/инсайты | Логика вычислений покрыта; графики проверяются глазами |

### Модули (вход → выход)

| Модуль | Вход | Выход |
|---|---|---|
| `pipeline` | Elering API (даты, зона LV) | `data/daily/*.json`, `data/aggregates.json` (коммит в repo) |
| `clock` | daily JSON (сегодня/завтра) | SVG-циферблат 24h с подсветкой текущего часа |
| `weekring` | aggregates (среднее по [день недели × час]) | SVG-кольцевая heatmap 7×24 |
| `spiral` | aggregates (среднее по месяцам) | SVG-спираль года |
| `insights` | daily + aggregates | 3–4 карточки с выводами (числа + текст) |

## 6. Монетизация *(формально: портфельный проект)*

Не применимо. Если бы развивался: free-виджет / Pro-алерты «дешёвый час начался»

## 7. Конкуренты

| «Конкурент» | Что делает | Чего не хватает (наше отличие) |
|---|---|---|
| Nord Pool / биржевые сайты | Таблицы, свечи | Нечитабельно для людей; ноль дизайна |
| Приложения энергокомпаний | Цена сейчас/завтра | Нет паттернов и цикличности; нельзя «увидеть неделю» |
| Типовой pet-dashboard (bar-charts) | Столбики на recharts | Не запоминается; наш — радиальный, custom D3 |
| dashboard.elering.ee | Официальные графики | Плоские линии; не про «когда включать стиралку» |

## 8. План запуска

### MVP (вечер 1–2)
- `pipeline`: скрипт fetch→normalize→JSON + workflow с cron; 60 дней истории
  бэконом загружены одним прогоном.
- `clock`: циферблат сегодня/завтра, подсветка текущего часа, tooltip, легенда.
- `insights`: 2 карточки («самый дешёвый час сегодня», «самый дорогой час»).
- Деплой на GitHub Pages.
- Метрика готовности: страница открывается по публичному URL, показывает
  **сегодняшние реальные** цены, обновляется без участия человека.

### v1.0 (вечер 3)
- `weekring` (7×24 heatmap) + `spiral` (сезонность).
- Ещё 2 insight-карточки; переключатель зоны LV/EE/LT (данные уже в API).
- Анимации входа (d3 transitions), полировка тултипов.
- README: скриншоты/GIF, «What this demonstrates», ссылка live.

### Полировка (по желанию)
- Vitest-тесты на агрегаты и инсайты; CI-бейдж.
- Lighthouse ≥ 90 (perf/accessibility) — цифра для README.

**Вне скоупа навсегда:** аккаунты, алерты/уведомления, бэкенд-сервер, мобильное
приложение, исторические данные глубже 1–2 лет.

## 9. Риски

| Риск | Вероятность | Митигация |
|---|---|---|
| CORS/недоступность Elering API из браузера | Средняя | Данные читаются НЕ из API в рантайме, а из закоммиченных JSON — фронту API не нужен |
| Смена формата API | Низкая | Нормализация в pipeline: фронт зависит только от своей JSON-схемы |
| «Красиво» съедает недели (перфекционизм) | Высокая | Дизайн-система зафиксирована в Техдеталях; после v1.0 — стоп |
| Радиальные графики нечитабельны при плохом исполнении | Средняя | Правила: цветовая шкала + легенда всегда; tooltip с точным числом; 00:00 сверху; сравнение только по углу/цвету, не по радиусу |
| Cron не сработал → данные устарели | Низкая | Фронт показывает дату снэпшота; бейдж «updated N hours ago» |

### Схемы данных (JSON)
```jsonc
// data/daily/YYYY-MM-DD.json
{ "date": "2026-07-05", "zone": "LV", "unit": "EUR/MWh",
  "prices": [ { "hour": 0, "price": 21.4 }, …24 ] }

// data/aggregates.json
{ "updated_at": "2026-07-05T15:04:00Z",
  "window_days": 60,
  "by_weekday_hour": [[…24]…7],   // средняя цена, Mon=0
  "by_month": [ { "month": 1, "avg": 96.2 }, …12 ] }
```
Цены хранить в EUR/MWh (как отдаёт API), в UI показывать **ц/кВт·ч** (= /10):
конверсия — одна функция в `lib/scales.ts`.

### Дизайн-система (зафиксирована, чтобы «красиво» не расползлось)
- **Тёмная тема**: фон #0B0E14; поверхность карточек #131826; текст #E6EAF2.
- **Шкала цены** (дёшево→дорого): #22C55E → #EAB308 → #EF4444; interpolate в d3.
- Акцент/подсветка текущего часа: #60A5FA. Радиус скругления карточек 16px.
- Типографика: Inter (заголовки 600, цифры tabular-nums).
- Правила радиальных графиков: 00:00 — сверху, ход по часовой; значение кодируется
  **длиной бара + цветом** (не только цветом); всегда легенда + tooltip с точным числом.
- Анимации: вход графика — 400–600 мс ease-out; hover — мгновенно.

### Пайплайн (не-AI)
`cron 15:00 EET → fetch Elering (вчера+сегодня+завтра, LV) → normalize →
пересчитать aggregates за 60 дней → commit → Pages redeploy`.
Бэкофилл истории: тот же скрипт с параметрами `--from --to` одним прогоном.

### Субагенты (Слой 4, превью)
`ai-agent-architect` не нужен. Команда:
- **frontend-developer** — React/D3-графики, UI (основная работа);
- **backend-engineer** — pipeline-скрипт и workflows;
- **qa-reviewer** (Read/Bash/Glob/Grep) — проверка схем JSON, корректности агрегатов
  (таймзона EET/EEST!), соответствия дизайн-системе.

⚠️ Особый риск для ревью: **таймзоны**. API отдаёт время в UTC; «час суток» для
циферблата и агрегатов считать в Europe/Riga, иначе паттерны съедут на 2–3 часа.
